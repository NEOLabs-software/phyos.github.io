Info

phyOS comes with pdwm, a dwm fork combining dwm and dwmblocks for a quick and efficent workspace. pdwm uses a shared library, and is easily configurable using the included Qt5 app. pdwm uses phyOS-picom, a picom fork for it's compositors. The terminal is called st, also known as simple terminal, with minimal distractions.

phyOS is a beautiful operating system. It's not like the others; it's vibrant and comfy. It's color scheme is Catppuccin, a soothing pastel theme for the high-spirited.

phyOS is free, and always will be. It is open source under the GNU GPLv2 license and can be found on GitHub.

About

phyOS was created by Arda Atci in 2022, because it was his one of his dreams for a while to make an operating system. Quote: "I really envy Torvalds (the creator of the Linux kernel), and it was my dream for 4 years to make an operating system. After those 4 years, that dream finally came true."

CUSTOMIZATION SCRIPTS

bm - bookmark script
Bookmark directories & navigate through them either from terminal or Win+d button combination

E.g: bm -a ./Desktop @Desktop will add a bookmark as @Desktop. Usage: jmp @Desktop from terminal, or Win+d then choose from rofi.

Actions:

  • bm -a $dir @Aliasname - Add bookmark
  • bm -r @Aliasname - Removes bookmark
  • bm -c - Removes all bookmarks
pOS-setbg
Set wallpaper according to ${img_path}, also update lock screen. If ${img_path} is not valid or empty, sets default wallpaper.
pOS-make-bar
Make your own status bar via script. Use vim keys or arrows to navigate. Hit enter to select a block, enter ---STOP--- to finish selection. Note that blocks will be ordered according to your selection order.
pOS-sddm-choose-theme
Choose from available sddm themes. Install custom themes with pacman. [Available themes in phyOS-repo]
pOS-grub-choose-theme
Choose from available grub themes. Install custom themes with pacman. [Available themes in phyOS-repo]

SYSTEM SCRIPTS

phyOS-grub-conf
Update grub configurations to phyOS latest
phyOS-update-kernel-conf
Run this script to update your kernel & initramfs configuration. If you have issues on boot, this will add a new kernel which early graphics (KMS) are disabled.
phyOS-update-mirrors
Run this script to update your mirrors to faster ones.
phyup - Update dotfiles & dwm script
Update dotfiles & dwm script. Run phyup dots --force to force update dotfiles to newest update.
pdwmc (pdwm control center)
Change attributes (colors, keys, buttons, app settings) for pdwm. Run pdwmc --help for available options.

phyOS-pdwm - dynamic window manager

back to wiki

DESCRIPTION

phyOS-pdwm is a fork of dwm for phyOS.

pdwm "orders" windows based on recency and primacy, while pdwm layouts may change, the most recent "master" window is shown in the most prominent position. There are bindings for cycling through and promoting windows to the master position.

Windows are grouped by tags. Each window can be tagged with one or multiple tags. Selecting certain tags displays all windows with these tags.

issues

Java applications which use the XToolkit/XAWT backend may draw grey windows only. The XToolkit/XAWT backend breaks ICCCM-compliance in recent JDK 1.5 and early JDK 1.6 versions, because it assumes a reparenting window manager. Possible workarounds are using JDK 1.4 (which doesn't contain the XToolkit/XAWT backend) or setting the environment variable .BR AWT_TOOLKIT=MToolkit (to use the older Motif backend instead) or running .B xprop -root -f _NET_WM_NAME 32a -set _NET_WM_NAME LG3D or .B wmname LG3D (to pretend that a non-reparenting window manager is running that the XToolkit/XAWT backend can recognize) or when using OpenJDK setting the environment variable
